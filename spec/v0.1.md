# Founder Passport · Specification v0.1

**Status**: Draft · 2026-04-24
**Author**: Fernando González (Cloudpany)
**License**: MIT (schema), proprietary (hosted implementation)

## 1. Design Principles

1. **W3C VC Data Model 2.0 base** — no new standards, layered schema on top of ratified foundations
2. **DIDs for identity** — `did:web` primary, `did:ebsi` for EU institutional credentials
3. **JSON-LD with versioned @context** — `https://schema.cloudpany.eu/founder-passport/v0.1`
4. **EdDSA signing default** (`eddsa-rdfc-2022`), ECDSA fallback
5. **StatusList2021 for revocation**
6. **SD-JWT for selective disclosure** of sensitive fields (revenue, valuation, equity %)
7. **Holder-controlled** — founder owns DID and keys, Cloudpany hosts but doesn't own
8. **EU-first** — PSD2 + eIDAS 2.0 + EUDI Wallet compatible from day 1

## 2. Passport Structure

The passport is a **container**, not itself a credential. It holds multiple signed credentials each issued independently.

```json
{
  "@context": [
    "https://www.w3.org/ns/credentials/v2",
    "https://schema.cloudpany.eu/founder-passport/v0.1"
  ],
  "type": ["FounderPassport"],
  "id": "did:web:cloudpany.eu:fp:fernando-gonzalez",
  "holder": "did:web:cloudpany.eu:fp:fernando-gonzalez",
  "version": "0.1",
  "created": "2026-05-15T10:30:00Z",
  "updated": "2026-05-20T14:12:00Z",
  "credentials": [ /* array of signed VCs */ ],
  "selfAttested": [ /* unverified claims marked explicitly */ ]
}
```

Rules:
- Holder controls the DID and signing key
- Each credential has independent issuer signature
- `selfAttested` is explicit and separate — never mixed with verified credentials
- Passport is exportable as JSON and re-importable in compatible implementations

## 5. EU-First Revenue Architecture (PSD2)

### Primary: PSD2 Open Banking via TrueLayer

- Covers 2500+ EU banks
- Legal right under PSD2/PSD3 — no per-bank negotiation
- Universal coverage regardless of payment processor
- Works for SaaS, e-commerce, services, B2B with SEPA transfers

## 6. eIDAS 2.0 Compatibility

### Format support required

- **W3C VC JSON-LD** (default) — our primary format
- **ISO/IEC 18013-5 mdoc** (future, for offline QR) — v1.0 roadmap
- **SD-JWT VC** (selective disclosure) — v0.1 for sensitive fields

### Transport protocols

- **OpenID4VCI** (Verifiable Credential Issuance) — required
- **OpenID4VP** (Verifiable Presentations) — required
- **SIOPv2** (Self-Issued OpenID) — optional

### Credential classification under eIDAS

| Credential | eIDAS tier | Path |
|---|---|---|
| IdentityCredential | **QEAA** | Import from EUDI Wallet / FNMT / Cl@ve |
| IncorporationCredential | **PubEAA** | Registros mercantiles (direct by 2027) |
| EducationCredential | **QEAA / PubEAA** | Europass + EUDI |
| LegalGoodStandingCredential | **PubEAA** | Registro mercantil |
| TeamSizeCredential | **PubEAA** (potential) | TGSS / equivalents |
| RevenueCredential | EAA | Private (PSD2 AISP licensed) |
| FundraisingCredential | EAA / QEAA via notary | Mixed |
| ExitCredential | EAA / QEAA via notary | Mixed |
| All others | EAA | Private issuers |

QEAA = Qualified Electronic Attestation of Attributes (highest legal weight)
PubEAA = Public Electronic Attestation of Attributes (public authority)
EAA = Electronic Attestation of Attributes (private, valid but not qualified)

### Compliance checklist (before v0.1 production)

- [ ] W3C VC Data Model 2.0 exact
- [ ] `eddsa-rdfc-2022` or `ecdsa-rdfc-2019` signing
- [ ] `did:web` resolution
- [ ] StatusList2021 revocation
- [ ] OpenID4VCI + OpenID4VP endpoints
- [ ] SD-JWT VC for sensitive fields
- [ ] Tested against EUDI Wallet Reference Implementation
- [ ] `.well-known/did.json` published and signed
- [ ] `.well-known/openid-credential-issuer` with trusted issuer list
- [ ] GDPR-compliant Privacy Policy
- [ ] Issuance logs with 10-year retention (eIDAS Article 24)

## 10. Open Questions / Deferred

- **Delegation**: passport signing on behalf of agent (v1.0, not MVP)
- **Zero-knowledge proofs**: beyond SD-JWT, explore BBS+ aggregation (v1.0)
- **Cross-passport relations**: co-founder attestations, mentor relationships (v0.2)
- **Revocation UX**: self-service founder + issuer + third-party dispute flows (v0.2)
- **EBSI registration**: complex EU process, defer to v1.0
